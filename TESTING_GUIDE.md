## 🐛 故障排查

### 问题 1：无法获取音频数据

**症状：** 日志中没有 `发送音频数据`

**解决方案：**
1. 检查麦克风权限
   - 设置 → 应用 → 权限 → 麦克风
2. 确保 QQ/微信 正在通话
3. 检查是否有声音输入
4. 重新启动应用

### 问题 2：音频源不是 VOICE_COMMUNICATION

**症状：** 日志显示 `MIC` 或 `VOICE_RECOGNITION`

**解决方案：**
1. 这是正常的降级行为
2. 说明设备不支持 VOICE_COMMUNICATION
3. 需要用户打开免提才能获取对方声音
4. 或者尝试其他通话应用

### 问题 3：WebSocket 连接失败

**症状：** 日志显示 `连接 WebSocket 失败`

**解决方案：**
1. 检查网络连接
2. 检查后端服务是否运行
3. 检查 WebSocket URL 是否正确
4. 查看详细错误信息

### 问题 4：语音识别不工作

**症状：** 没有 `临时识别` 或 `最终识别` 日志

**解决方案：**
1. 检查百度 API 配置
2. 确保有网络连接
3. 检查是否有声音输入
4. 查看详细错误信息

### 问题 5：应用崩溃

**症状：** 应用突然关闭

**解决方案：**
1. 查看完整的错误日志
2. 检查权限是否已授予
3. 尝试 `flutter clean && flutter run`
4. 检查是否有内存泄漏

## 📈 性能监控

### 监控指标

1. **CPU 使用率**
   - 正常：< 20%
   - 警告：20-50%
   - 严重：> 50%

2. **内存使用**
   - 正常：< 100MB
   - 警告：100-200MB
   - 严重：> 200MB

3. **网络带宽**
   - 音频：~16 KB/s
   - 视频：~40 KB/s
   - 文字：~1 KB/s
   - 总计：~57 KB/s

4. **电池消耗**
   - 正常：< 5% per hour
   - 警告：5-10% per hour
   - 严重：> 10% per hour

### 查看性能

使用 Android Studio 的 Profiler：
1. 打开 Android Studio
2. 连接设备
3. 点击 Profiler 标签
4. 观察 CPU、内存、网络使用情况
