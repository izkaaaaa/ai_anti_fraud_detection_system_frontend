## 📡 API 对接说明

### 家庭组管理 API

| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 创建家庭组 | `/api/family/create?name={name}` | POST | 创建者自动成为管理员 |
| 申请加入 | `/api/family/{family_id}/apply` | POST | 需等待管理员审批 |
| 查看申请列表 | `/api/family/applications` | GET | 仅管理员可访问 |
| 审批申请 | `/api/family/applications/{app_id}?is_approve={bool}` | PUT | 仅管理员可操作 |

### 安全报告 API

| 功能 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 生成报告 | `/api/users/{user_id}/security-report` | GET | 调用大模型生成 Markdown 报告 |

## 🔧 使用说明

### 1. 家庭组管理

**创建家庭组：**
1. 进入"家庭组"页面
2. 点击"创建家庭组"按钮
3. 输入家庭组名称
4. 创建成功后，自动成为管理员

**加入家庭组：**
1. 进入"家庭组"页面
2. 点击"加入家庭组"按钮
3. 输入家庭组ID
4. 等待管理员审批

**审批申请（管理员）：**
1. 进入"家庭组"页面
2. 查看"待审批申请"列表
3. 点击"同意"或"拒绝"按钮

---

### 2. 安全报告

**生成报告：**
1. 进入"我的"页面
2. 点击"安全报告"菜单项（带 NEW 标签）
3. 点击"生成报告"按钮
4. 等待 AI 分析（3-10秒）
5. 查看 Markdown 格式的报告

**刷新报告：**
- 点击右上角刷新按钮

## ⚠️ 注意事项

### 1. 后端依赖

**家庭组功能：**
- 需要后端执行数据库迁移：`alembic upgrade head`
- 新增了 `family_groups` 和 `family_applications` 表
- `users` 表新增了 `family_id` 字段

**安全报告功能：**
- 需要配置大模型 API（在 `.env` 文件中）
- 需要配置：`LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL_NAME`
- 如果大模型未配置，接口会返回错误提示

### 2. 权限控制

- 查看申请列表：仅管理员
- 审批申请：仅管理员
- 生成安全报告：需要登录

### 3. 用户体验

- 安全报告生成需要 3-10 秒，已添加加载动画
- 家庭组ID可以复制，方便分享
- 所有操作都有成功/失败提示
