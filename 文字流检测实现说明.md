# 文字流检测实现说明

## ✅ 已完成的修改

### 1. RealTimeDetectionService.dart

**新增功能：**
- 集成了 `BaiduSpeechService`（百度实时语音识别）
- 在 `startDetection()` 中启动语音识别
- 在 `stopDetection()` 中停止语音识别
- 语音识别结果自动发送给后端进行文本检测

**关键代码：**
```dart
// 启动语音识别
final speechStarted = await _startSpeechRecognition();

// 监听识别结果并发送给后端
_speechService.onFinalResult = (text, startTime, endTime) {
  if (text.isNotEmpty && _isConnected && _channel != null) {
    sendText(text);  // 发送文本给后端检测
  }
};
```

```
用户点击"开始监测"
    ↓
1. 创建通话记录 ✓
    ↓
2. 连接 WebSocket ✓
    ↓
3. 启动音频录制 ✓ (用于音频检测)
    ↓
4. 启动视频采集 ✓ (用于视频检测)
    ↓
5. 启动语音识别 ✓ (用于文字检测) ← 新增
    ↓
实时监测中：
  - 每3秒发送音频数据 → 后端音频检测
  - 每秒发送视频帧 → 后端视频检测
  - 实时识别语音 → 转文字 → 后端文本检测 ← 新增
    ↓
接收检测结果：
  - audio_result (音频检测结果)

    ↓
显示风险警告（如果检测到风险）
```

## 📊 三种检测方式对比

| 检测类型 | 数据来源 | 发送频率 | 检测内容 |
|---------|---------|---------|---------|
| **音频检测** | 麦克风录音 | 每3秒 | AI合成语音、变声器 |
| **视频检测** | 摄像头采集 | 每秒1-5帧 | Deepfake换脸 |
| **文本检测** | 语音识别 | 实时（每句话） | 诈骗话术关键词 |

## 🧪 测试方法

### 测试文字流检测

1. **启动应用**
   ```bash
   flutter run
   ```

2. **进入实时监测页面**
   - 点击底部导航栏的"监测"

3. **开始监测**
   - 点击"开始监测"按钮
   - 授予麦克风和摄像头权限

4. **说话测试**
   - 对着手机说话（例如："你好，测试一下"）
   - 观察日志输出：
     ```
     🎤 临时识别: 你好
     ✅ 最终识别: 你好，测试一下 (1000ms - 3000ms)
     📝 发送文本数据: 你好，测试一下
     ```

5. **说诈骗话术测试**
   - 说包含诈骗关键词的话（例如："请把验证码告诉我"）
   - 观察是否收到文本风险警告
   - 检查是否显示关键词标签（"验证码"）

6. **查看检测结果**
   - 在"检测结果"卡片中查看"文本检测"
   - 如果检测到风险，会显示：
     - 红色边框
     - 风险标签
     - 关键词列表

## ⚠️ 注意事项

1. **权限要求**
   - 麦克风权限（必需）
   - 摄像头权限（可选）

2. **网络要求**
   - 需要连接到后端服务器
   - 需要访问百度语音识别 API

3. **性能考虑**
   - 语音识别会增加一些 CPU 和网络开销
   - 如果设备性能较差，可以考虑降低视频帧率

4. **错误处理**
   - 如果语音识别启动失败，不会影响音视频检测
   - 会在日志中输出警告信息

## 🐛 常见问题

### Q1: 语音识别没有启动？
**A:** 检查以下几点：
- 是否授予了麦克风权限
- 网络是否正常（需要访问百度 API）
- 查看日志是否有错误信息

### Q2: 说话了但没有识别结果？
**A:** 可能的原因：
- 环境噪音太大
- 说话声音太小
- 网络延迟较高
- 百度 API 配额用完

### Q3: 文本检测结果不准确？
**A:** 这取决于：
- 语音识别的准确度
- 后端文本检测模型的准确度
- 建议说话清晰、标准普通话
