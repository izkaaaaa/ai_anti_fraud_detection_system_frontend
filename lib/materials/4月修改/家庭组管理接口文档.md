# 家庭组管理接口文档

## 一、数据模型

### 1.1 家庭组（FamilyGroup）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 家庭组 ID |
| group_name | string | 家庭组名称 |
| admin_id | int | 主管理员用户 ID |
| created_at | datetime | 创建时间 |

### 1.2 家庭组成员（User.family_id）

普通成员在 User 表中通过 `family_id` 字段关联家庭组。

### 1.3 管理员角色（FamilyAdmin）

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | int | 管理员用户 ID |
| family_id | int | 家庭组 ID |
| admin_role | string | 角色：`primary`（主管理员）/ `secondary`（副管理员） |

### 1.4 申请记录（FamilyApplication）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 申请记录 ID |
| family_id | int | 申请加入的家庭组 ID |
| user_id | int | 申请人用户 ID |
| status | enum | `pending`（待审批）/ `approved`（已同意）/ `rejected`（已拒绝） |
| created_at | datetime | 申请时间 |

### 1.5 角色权限说明

| 角色 | 说明 |
|------|------|
| `primary`（主管理员） | 家庭组创建者，拥有最高权限：审批加入申请、设置管理员角色、移除成员、转让主管理员、解散家庭组 |
| `secondary`（副管理员） | 主管理员任命，可以审批加入申请、移除普通成员、远程干预被监护人 |
| `member`（普通成员） | 被保护对象，可以主动发起求助（仅SOS）、退出家庭组 |

## 三、接口详情

### 3.1 创建家庭组

创建家庭组，创建者自动成为主管理员。

**请求**

```
POST /api/family/create?name=<家庭组名称>
```

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 家庭组名称 |

**响应**

```json
{
  "code": 200,
  "message": "家庭组创建成功",
  "data": {
    "family_id": 1,
    "group_name": "王家防骗组"
  }
}
```

**业务规则**

- 创建者自动成为主管理员（`admin_role: primary`）
- 创建者本人不在家庭组成员中（`family_id` 保持为 `None`），是管理者而非被保护对象
- 已加入其他家庭组的普通成员不能再创建家庭组
- 有待审批申请的用户需先取消申请

---

### 3.2 申请加入家庭组

用户提交加入家庭组的申请，等待管理员审批。

**请求**

```
POST /api/family/{family_id}/apply
```

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| family_id | int | 要申请加入的家庭组 ID |

**响应**

```json
{
  "code": 200,
  "message": "申请已发送，等待管理员审批"
}
```

**业务规则**

- 已有家庭组的用户不能再申请加入新家庭组（一人只能在一个家庭组）
- 不能重复申请（已有 pending 状态申请时返回 400）

---

### 3.3 获取待审批列表（管理员端）

查询当前用户作为管理员的所有家庭组中的待审批申请。

**请求**

```
GET /api/family/applications
```

**响应**

```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "items": [
      {
        "application_id": 1,
        "family_id": 1,
        "family_name": "王家防骗组",
        "user_id": 10,
        "username": "小王",
        "phone": "13800138000",
        "role_type": "elder",
        "apply_time": "2026-04-05 10:00:00"
      }
    ]
  }
}
```

**业务规则**

- 非管理员调用返回 403
- 只返回当前用户管理的所有家庭组的 pending 申请

---

### 3.4 获取我的申请列表（用户端）

查询当前用户提交的所有家庭组申请。

**请求**

```
GET /api/family/my-applications
```

**响应**

```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "items": [
      {
        "application_id": 1,
        "family_id": 1,
        "family_name": "王家防骗组",
        "status": "pending",
        "apply_time": "2026-04-05 10:00:00"
      }
    ]
  }
}
```

**status 可选值**：`pending` / `approved` / `rejected`

---

### 3.5 取消我的申请

取消处于 pending 状态的申请。

**请求**

```
DELETE /api/family/applications/{app_id}
```

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| app_id | int | 申请记录 ID |

**响应**

```json
{
  "code": 200,
  "message": "申请已取消"
}
```

**业务规则**

- 仅申请人本人可以取消
- 仅 pending 状态的申请可以取消

---

### 3.6 审批申请（管理员）

管理员同意或拒绝用户的加入申请。

**请求**

```
PUT /api/family/applications/{app_id}?is_approve=<true|false>
```

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| app_id | int | 申请记录 ID |

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| is_approve | bool | 是 | `true` = 同意，`false` = 拒绝 |

**响应（同意）**

```json
{
  "code": 200,
  "message": "操作成功，已同意"
}
```

**响应（自动拒绝：申请人已有家庭组）**

```json
{
  "code": 400,
  "message": "该用户已加入其他家庭组，无法批准此申请",
  "data": {
    "auto_rejected": true,
    "reason": "user_already_in_family",
    "current_family_id": 5
  }
}
```

**业务规则**

- 非管理员无权操作
- 已处理过的申请（非 pending）不能再操作
- 申请人已有家庭组时自动拒绝

---

### 3.7 获取家庭成员列表

**请求**

```
GET /api/family/members?family_id=<family_id>
```

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| family_id | int | 否 | 家庭组 ID，不传则查当前用户所在组 |

**响应**

```json
{
  "code": 200,
  "message": "获取成员列表成功",
  "data": {
    "family_id": 1,
    "group_name": "王家防骗组",
    "members": [
      {
        "user_id": 10,
        "username": "小王",
        "name": "王明",
        "phone": "13800138000",
        "role_type": "elder",
        "admin_role": "none",
        "is_me": false
      },
      {
        "user_id": 20,
        "username": "老王",
        "name": "王强",
        "phone": "13900139000",
        "role_type": "guardian",
        "admin_role": "primary",
        "is_me": true
      }
    ]
  }
}
```

**admin_role 取值**：`primary`（主管理员）/ `secondary`（副管理员）/ `none`（普通成员）

**业务规则**

- 不传 `family_id` 时，自动取当前用户的 `family_id`
- 传入 `family_id` 时，必须是成员或管理员才能查看

---

### 3.8 设置/取消管理员（主管理员）

主管理员将某成员设为副管理员，或取消其管理员身份。

**请求**

```
PUT /api/family/members/{user_id}/admin-role?role=<none|secondary|primary>
```

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| user_id | int | 目标用户 ID |

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| role | string | 是 | 目标角色：`none`（取消管理员）/ `secondary`（副管理员）/ `primary`（主管理员） |

**响应**

```json
{
  "code": 200,
  "message": "已将 小王 设置为副管理员"
}
```

**业务规则**

- 只有主管理员可以操作
- 设置为主管理员时，原主管理员自动降级为副管理员
- 家庭组的 `admin_id` 同时更新为新的主管理员
- 取消管理员（`none`）时，如果用户也是其他家庭组的管理员，则 `is_admin` 保持为 `true`

---

### 3.9 移除成员（管理员）

管理员将某成员移出家庭组。

**请求**

```
DELETE /api/family/members/{user_id}
```

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| user_id | int | 要移除的用户 ID |

**响应**

```json
{
  "code": 200,
  "message": "已将该成员移出家庭组"
}
```

**业务规则**

- 管理员可以移除普通成员和副管理员
- 主管理员不能被移除（只能主动退出或转让）
- 副管理员不能移除其他副管理员
- 被移除成员的 `family_id` 置为 `None`，管理员记录同步删除

---

### 3.10 退出家庭组

普通成员、副管理员、主管理员各自退出。

**请求**

```
POST /api/family/leave
```

**响应**

| 场景 | code | message |
|------|------|---------|
| 普通成员退出 | 200 | 已退出家庭组 |
| 副管理员退出 | 200 | 已退出家庭组 |
| 主管理员退出（有其他成员） | 400 | 您是主管理员，家庭组内还有其他成员... |
| 主管理员退出（只剩自己） | 200 | 家庭组已解散 |

**业务规则**

- 主管理员退出时：如果还有其他成员，必须先转让主管理员或移除所有成员
- 主管理员退出时：如果只剩自己一人，自动解散整个家庭组（删除组、申请记录、管理员记录）

### 3.13 一键通报（SOS）

被监护人主动向监护人发送求助信号。

**请求**

```
POST /api/family/sos?call_id=<call_id>&message=<求助信息>
```

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| call_id | int | 是 | 当前通话 ID |
| message | string | 否 | 求助信息，默认"我正在遭遇可疑通话，请立即联系我！" |

**响应**

```json
{
  "code": 200,
  "message": "求助信号已发送给 2 位监护人",
  "data": {
    "notified_count": 2,
    "call_id": 123,
    "timestamp": "2026-04-05T10:30:00"
  }
}
```

**监护人收到的 WebSocket 消息（sos_alert）**

```json
{
  "type": "sos_alert",
  "data": {
    "title": "紧急求助",
    "message": "您的家人【张三】正在请求帮助！我正在遭遇可疑通话，请立即联系我！",
    "victim_id": 10,
    "victim_name": "张三",
    "victim_phone": "138****8888",
    "call_id": 123,
    "family_id": 1,
    "timestamp": "2026-04-05T10:30:00",
    "display_mode": "popup",
    "action": "vibrate",
    "urgency": "high"
  }
}
```

**业务规则**

- 未加入家庭组的用户无法使用
- 求助信号只推送给监护人（管理员），不推送给其他普通成员

---

### 3.14 远程干预（监护人）

监护人远程控制被监护人的通话。

**请求**

```
POST /api/family/remote-intervene?target_user_id=<user_id>&action=<action>&message=<message>
```

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| target_user_id | int | 是 | 被干预的用户 ID |
| action | string | 是 | 干预动作：`block_call`（强制挂断）/ `warn`（发送警告）/ `check_status`（检查状态） |
| message | string | 否 | 附加消息，warn 动作时使用 |

**响应**

```json
{
  "code": 200,
  "message": "干预指令已发送给 张三",
  "data": {
    "action": "warn",
    "target_user_id": 10,
    "target_user_name": "张三",
    "timestamp": "2026-04-05T10:35:00"
  }
}
```

**被监护人收到的 WebSocket 消息（remote_control）**

> action = `block_call` 时：

```json
{
  "type": "remote_control",
  "data": {
    "action": "block_call",
    "from_admin_id": 20,
    "from_admin_name": "老王",
    "target_user_id": 10,
    "message": "监护人 老王 已为您强制挂断此通话",
    "timestamp": "2026-04-05T10:35:00",
    "control": {
      "block_call": true,
      "warning_mode": "fullscreen",
      "ui_message": "监护人 老王 已为您强制挂断此通话"
    }
  }
}
```

> action = `warn` 时：

```json
{
  "type": "remote_control",
  "data": {
    "action": "warn",
    "from_admin_id": 20,
    "from_admin_name": "老王",
    "target_user_id": 10,
    "message": "注意安全，谨防诈骗！",
    "timestamp": "2026-04-05T10:35:00",
    "control": {
      "block_call": false,
      "warning_mode": "popup",
      "ui_message": "监护人 老王 提醒您注意通话安全"
    }
  }
}
```

> action = `check_status` 时：

```json
{
  "type": "remote_control",
  "data": {
    "action": "check_status",
    "from_admin_id": 20,
    "from_admin_name": "老王",
    "target_user_id": 10,
    "message": "",
    "timestamp": "2026-04-05T10:35:00",
    "control": {
      "request_status": true
    }
  }
}
```

收到 `check_status` 指令后，前端需要主动上报当前状态，调用监护人查询状态接口或通过 WebSocket 回复。

**业务规则**

- 操作者必须是目标用户所在家庭组的管理员
- 操作者与目标用户必须在同一家庭组

## 四、特殊 WebSocket 消息汇总

家庭组相关功能会通过 WebSocket 向监护人推送以下消息：

| type | 触发场景 | 接收方 |
|------|---------|--------|
| `sos_alert` | 成员发起一键通报 | 家庭组所有管理员 |
| `family_alert` | 成员通话被检测到诈骗风险 | 家庭组所有管理员 |
| `emergency_alert` | 成员触发一键报警 | 家庭组所有管理员 |
| `remote_control` | 监护人远程干预指令 | 被监护人用户 |

详见 [前端 WebSocket 实时消息格式文档](./前端WebSocket实时消息格式文档.md)。
