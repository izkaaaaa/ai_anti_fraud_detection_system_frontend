# AI反诈骗系统 - 新增接口文档

> **版本说明**：本文档记录了家庭组管理和智能安全报告生成的新增接口

---

## 一、家庭组管理接口

### 1.1 创建家庭组 🔒

**接口**: `POST /api/family/create`  
**认证**: 需要 JWT Token  
**请求头**: `Authorization: Bearer {access_token}`

**请求参数**:
```
URL Query 参数:
- name: string (必填) - 家庭组名称
```

**响应示例**:
```json
{
  "code": 200,
  "message": "家庭组创建成功",
  "data": {
    "family_id": 1,
    "group_name": "我的家庭"
  }
}
```

**说明**:
- 创建者自动成为该家庭组的管理员
- 创建者会自动绑定到该家庭组
- 一个用户只能在一个家庭组中

---

### 1.2 申请加入家庭组 🔒

**接口**: `POST /api/family/{family_id}/apply`  
**认证**: 需要 JWT Token  
**请求头**: `Authorization: Bearer {access_token}`

**URL 参数**:
```
- family_id: integer (必填) - 要加入的家庭组ID
```

**请求体**: 无

**响应示例**:
```json
{
  "code": 200,
  "message": "申请已发送，等待管理员审批"
}
```

**错误响应**:
```json
// 家庭组不存在
{
  "code": 404,
  "message": "未找到该ID的家庭组"
}

// 重复申请
{
  "code": 400,
  "message": "您已提交过申请，请等待管理员审批"
}
```

**说明**:
- 用户输入家庭组ID后调用此接口
- 申请会进入待审批状态
- 需要等待家庭组管理员审批

---

### 1.3 获取待审批申请列表 🔒 (管理员专用)

**接口**: `GET /api/family/applications`  
**认证**: 需要 JWT Token (必须是家庭组管理员)  
**请求头**: `Authorization: Bearer {access_token}`

**请求体**: 无

**响应示例**:
```json
{
  "code": 200,
  "message": "获取成功",
  "data": [
    {
      "application_id": 1,
      "user_id": 5,
      "phone": "13800138000",
      "apply_time": "2026-03-03 10:30:00"
    },
    {
      "application_id": 2,
      "user_id": 6,
      "phone": "13900139000",
      "apply_time": "2026-03-03 11:00:00"
    }
  ]
}
```

**错误响应**:
```json
{
  "code": 403,
  "message": "无权查看，您不是家庭组管理员"
}
```

**说明**:
- 只有家庭组管理员可以调用
- 返回所有待审批的申请
- 包含申请人的基本信息

---

### 1.4 审批加入申请 🔒 (管理员专用)

**接口**: `PUT /api/family/applications/{app_id}`  
**认证**: 需要 JWT Token (必须是家庭组管理员)  
**请求头**: `Authorization: Bearer {access_token}`

**URL 参数**:
```
- app_id: integer (必填) - 申请记录ID
```

**Query 参数**:
```
- is_approve: boolean (必填) - true=同意，false=拒绝
```

**请求示例**:
```
PUT /api/family/applications/1?is_approve=true
```

**响应示例**:
```json
// 同意
{
  "code": 200,
  "message": "操作成功，已同意"
}

// 拒绝
{
  "code": 200,
  "message": "操作成功，已拒绝"
}
```

**错误响应**:
```json
// 无效申请
{
  "code": 404,
  "message": "无效的申请或已处理"
}

// 无权限
{
  "code": 403,
  "message": "无权审批此申请"
}
```

**说明**:
- 只有家庭组管理员可以审批
- 同意后，申请人会自动加入家庭组
- 拒绝后，申请状态变为已拒绝

## 二、智能安全报告接口

### 2.1 生成用户安全监测报告

**接口**: `GET /api/users/{user_id}/security-report`  
**认证**: 可选 (建议需要认证)  
**请求头**: `Authorization: Bearer {access_token}` (可选)

**URL 参数**:
```
- user_id: integer (必填) - 用户ID
```

**请求体**: 无

**响应示例**:
```json
{
  "user_id": 1,
  "username": "zhangsan",
  "report_generated_at": "2026-03-03T14:30:00",
  "report_content": "## 个人反诈安全监测报告\n\n### 📊 核心评估结论\n\n您的综合安全评级：**良好** ✅\n\n近期监测显示，您的防诈意识较强，但仍需保持警惕...\n\n### 📈 近期风险数据回顾\n\n- 监测时间段：2026-02-20 至 2026-03-03\n- 总通话次数：10 次\n- 拦截高风险通话：2 次\n- 主要诈骗类型：冒充公检法、虚假理财\n\n### ⚠️ 暴露的薄弱点分析\n\n根据您的通话记录分析，发现以下风险点：\n\n1. **对"公检法"话术警惕性不足**：在2月28日的通话中，对方自称是"市公安局民警"，您未能第一时间挂断...\n\n### 💡 专属防骗建议\n\n针对您的【老人】角色特征，我们为您定制以下建议：\n\n1. **牢记"三不原则"**：\n   - 不轻信陌生来电\n   - 不透露个人信息\n   - 不转账汇款\n\n2. **遇到以下情况立即挂断**：\n   - 自称公检法要求转账的\n   - 推销特效药、保健品的\n   - 声称中奖需要先交税的\n\n3. **启用家庭守护功能**：\n   - 建议绑定子女为监护人\n   - 高风险通话会实时通知家人\n\n---\n\n**报告生成时间**：2026-03-03 14:30:00  \n**下次更新时间**：建议每周查看一次"
}
```

**说明**:
- 调用大模型分析用户近期通话记录（最多10条）
- 生成 Markdown 格式的可视化报告
- 报告内容包括：
  - 综合安全评级
  - 近期风险数据统计
  - 薄弱点分析
  - 个性化防骗建议（根据用户角色类型定制）
- 报告内容可直接在前端使用 Markdown 渲染器展示

**前端使用建议**:
```javascript
// 获取报告
const response = await fetch(`/api/users/${userId}/security-report`, {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const data = await response.json();

// 使用 Markdown 渲染库展示
// 例如：marked.js, react-markdown 等
const htmlContent = marked.parse(data.report_content);
```

## 三、数据库表结构变更

### 3.1 家庭组表 (family_groups)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int | 主键，家庭组ID |
| group_name | varchar(100) | 家庭组名称 |
| owner_id | int | 创建者/管理员ID (外键关联 users.user_id) |
| created_at | datetime | 创建时间 |

### 3.2 家庭组申请表 (family_applications)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int | 主键，申请记录ID |
| family_id | int | 家庭组ID (外键关联 family_groups.id) |
| user_id | int | 申请人ID (外键关联 users.user_id) |
| status | enum | 申请状态：pending(待审批)、approved(已同意)、rejected(已拒绝) |
| created_at | datetime | 申请时间 |

## 四、业务流程说明

### 4.1 家庭组创建与加入流程

```
1. 用户A创建家庭组
   ↓
   POST /api/family/create?name=我的家庭
   ↓
   返回 family_id: 1
   ↓
   用户A自动成为管理员并加入

2. 用户B申请加入
   ↓
   用户B输入家庭组ID: 1
   ↓
   POST /api/family/1/apply
   ↓
   申请进入待审批状态

3. 用户A审批申请
   ↓
   GET /api/family/applications (查看待审批列表)
   ↓
   PUT /api/family/applications/1?is_approve=true
   ↓
   用户B加入成功
```

### 4.2 安全报告生成流程

```
1. 前端请求生成报告
   ↓
   GET /api/users/1/security-report
   ↓
2. 后端查询用户信息和近期通话记录
   ↓
3. 调用大模型 (LLM) 分析
   ↓
4. 生成 Markdown 格式报告
   ↓
5. 返回给前端
   ↓
6. 前端使用 Markdown 渲染器展示
```

## 五、注意事项

### 5.1 权限控制
- 创建家庭组：任何登录用户
- 申请加入：任何登录用户
- 查看申请列表：仅家庭组管理员
- 审批申请：仅家庭组管理员
- 生成安全报告：建议需要认证（可以是本人或家庭成员）

### 5.2 业务限制
- 一个用户只能在一个家庭组中
- 一个家庭组只有一个管理员（创建者）
- 重复申请会被拒绝
- 已处理的申请不能再次审批

### 5.3 大模型配置
- 安全报告生成依赖大模型服务
- 需要在 `.env` 文件中配置：
  ```
  LLM_API_KEY=your_api_key
  LLM_BASE_URL=https://api.openai.com/v1
  LLM_MODEL_NAME=gpt-4
  ```
- 如果大模型调用失败，会返回默认错误提示

### 5.4 性能优化建议
- 安全报告生成可能需要 3-10 秒（取决于大模型响应速度）
- 建议前端显示加载动画
- 可以考虑缓存报告结果（例如：每天只生成一次）
