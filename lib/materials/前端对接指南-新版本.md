# 前端对接指南 - 新版本

## 📋 本次更新概述

后端新增了用户画像字段，用于 LLM 个性化分析和风险评估。

---

## 🆕 新增字段说明

### 用户模型新增字段

| 字段名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| `gender` | string | 否 | 性别 | "男" / "女" / "未知" |
| `profession` | string | 否 | 职业 | "工程师" / "教师" / "学生" |
| `marital_status` | string | 否 | 婚姻状况 | "单身" / "已婚" / "离异" |

---

## 🔄 接口变更

### 1. 用户注册接口（已更新）

**接口**: `POST /api/users/register`

**新增可选字段**:

```javascript
// 旧版本
{
  "phone": "13800138000",
  "username": "testuser",
  "name": "张三",
  "password": "123456",
  "sms_code": "666666",
  "role_type": "青壮年"
}

// 新版本（新增三个可选字段）
{
  "phone": "13800138000",
  "username": "testuser",
  "name": "张三",
  "password": "123456",
  "sms_code": "666666",
  "role_type": "青壮年",
  "gender": "男",              // 新增
  "profession": "工程师",       // 新增
  "marital_status": "已婚"     // 新增
}
```

---

### 3. 用户画像更新接口（新增）

**接口**: `PUT /api/users/profile` 🔒

**说明**: 用户可以更新自己的画像信息

**请求头**:
```
Authorization: Bearer {access_token}
```

**请求体**:
```json
{
  "role_type": "老人",
  "gender": "女",
  "profession": "退休教师",
  "marital_status": "已婚"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "用户画像更新成功",
  "data": {
    "user_id": 1,
    "role_type": "老人"
  }
}
```

**UI 建议**:

- 可以在注册后引导用户完善

### 2. 个人中心页面

**新增"完善资料"功能**:

```javascript
// 更新用户画像
async function updateProfile(data) {
  const response = await fetch('/api/users/profile', {
    method: 'PUT',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify({
      role_type: data.role_type,

    })

```

---

### 3. 安全报告页面

**说明**: 这些字段会影响 LLM 生成的安全报告内容

- 如果用户填写了 `gender`、`profession`、`marital_status`
- LLM 会根据这些信息生成更个性化的防骗建议

**示例**:
- 职业为"学生" → 提示警惕"兼职刷单"、"校园贷"
- 婚姻状况为"单身" → 提示警惕"杀猪盘"、"婚恋诈骗"
- 性别为"女" → 提示警惕"美容贷"、"网购退款"

## 🔧 兼容性说明

### 向后兼容

✅ **旧版本前端仍然可以正常使用**

- 新增字段都是可选的
- 不传这些字段不会报错
- 已有功能不受影响

### 建议升级

虽然兼容旧版本，但建议前端升级以获得更好的用户体验：

1. **注册时收集更多信息** → LLM 分析更准确
2. **提供"完善资料"入口** → 提升用户画像完整度
3. **安全报告更个性化** → 提高用户满意度

## 🚨 注意事项

1. **字段值建议使用固定选项**，避免用户随意输入
2. **性别字段**：建议只提供"男"、"女"、"未知"三个选项
3. **婚姻状况**：建议只提供"单身"、"已婚"、"离异"三个选项
4. **职业字段**：可以提供常见职业列表供选择，也可以自由输入
5. **这些字段会影响 LLM 分析结果**，建议引导用户真实填写
