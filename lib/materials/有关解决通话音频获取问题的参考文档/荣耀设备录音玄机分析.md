# 荣耀设备通话录音玄机分析

> 基于对通话录音Pro源码的深度逆向分析，揭示荣耀设备上通话录音成功的真正原因。

---

## 现象描述

在荣耀手机上使用通话录音Pro时，有以下奇特表现：

1. QQ/微信通话时，通话录音Pro可以在后台录到声音
2. 点击"开始录音"后，**无法切回通话录音Pro主界面**
3. 屏幕上只留一个悬浮窗，QQ通话界面始终保持在前台
4. 但录音一直在进行

这不是bug，是精心设计的机制。

---

## 核心机制拆解

### 第一层：无障碍服务持续监听

`RecordHelperService.onAccessibilityEvent()` 中，针对荣耀/华为设备有专门逻辑：

```java

if (TextUtils.equals(str.toLowerCase(), "huawei") ||
    TextUtils.equals(str.toLowerCase(), "honor")) {

    if (z) {
        // 正在录音时：每次有任何窗口事件，都重新启动 OnePXActivity
        Intent intent = new Intent(this, OnePXActivity.class);
        last.startActivity(intent);
    } else if (last instanceof OnePXActivity) {
        // 没在录音时：销毁 OnePXActivity
        last.finish();
    }
}
```

关键点：**`onAccessibilityEvent` 在任何窗口切换时都会触发**，也就是说只要录音在进行，每次窗口变化都会重新启动 `OnePXActivity`。

### 第三层：完整的闭环机制

```
录音开始 (f7645OooOOOo = true)
        ↓
用户尝试切换任何窗口
        ↓
onAccessibilityEvent 触发
        ↓
检测到荣耀设备 + 正在录音
        ↓
启动 OnePXActivity（1像素）
        ↓
OnePXActivity 检测到正在录音
        ↓
500ms 后执行 performGlobalAction(GLOBAL_ACTION_BACK)
        ↓
系统模拟按下返回键
        ↓
用户被打回上一个界面（QQ通话）
        ↓
触发窗口切换事件
        ↓
onAccessibilityEvent 再次触发（回到第2步，形成闭环）
```

**这就是为什么在录音期间，荣耀设备上永远切不回通话录音Pro主界面。**

---

### 第四层：为什么这样能录到声音

- 录音服务用的是 `AudioSource.VOICE_COMMUNICATION`（值=6）
- 荣耀系统对后台应用访问 `VOICE_COMMUNICATION` 有限制
- 但**无障碍服务**本身是系统级服务，不受后台限制
- `OnePXActivity` 虽然1像素，但它是一个真实存在的前台Activity，让系统认为应用有前台界面
- **前台Activity + 无障碍服务 + 前台通知** 三重保活，使荣耀系统允许后台录音服务持续访问 `VOICE_COMMUNICATION`

---

## 为什么只剩悬浮窗

悬浮窗（`WindowManager` 添加的 View）不属于任何 Activity，它是系统级浮层：
- 不会触发 `onAccessibilityEvent` 的窗口切换事件
- 不受 `performGlobalAction(BACK)` 影响
- 可以悬浮在任何应用上方

所以悬浮窗成了录音期间唯一的用户交互入口，用户只能通过悬浮窗停止录音，无法通过切换界面干扰录音流程。

## 对我们Flutter项目的启示

### 必须实现的组件

#### 1. OnePxActivity（荣耀/华为专用）

```kotlin
class OnePxActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        if (!isRecording()) {
            // 没在录音，直接回主界面
            startActivity(Intent(this, MainActivity::class.java))
            finish()
            return
        }

        // 缩成1像素
        val window = window
        window.setGravity(Gravity.LEFT or Gravity.TOP)
        val params = window.attributes
        params.x = 0
        params.y = 0
        params.width = 1
        params.height = 1
        window.attributes = params

        // 500ms后模拟返回键
        Handler(Looper.getMainLooper()).postDelayed({
            performGlobalActionBack()
        }, 500)
    }

    private fun performGlobalActionBack() {
        // 通过MethodChannel通知无障碍服务执行返回键
        // 或者直接在无障碍服务中执行
    }
}
```

#### 2. 无障碍服务中的荣耀设备逻辑

```kotlin
override fun onAccessibilityEvent(event: AccessibilityEvent?) {
    val isRecording = RecordingManager.isRecording
    val brand = Build.BRAND.lowercase()

    // 荣耀/华为专属处理
    if (brand == "huawei" || brand == "honor") {
        if (isRecording) {
            // 每次窗口切换，启动1像素Activity
            val intent = Intent(this, OnePxActivity::class.java)
            intent.flags = Intent.FLAG_ACTIVITY_NEW_TASK
            startActivity(intent)
        }
    }

    // 通用：检测QQ/微信通话
    // ...
}
```

#### 3. 悬浮窗作为唯一交互入口

录音期间必须有悬浮窗显示录音状态和停止按钮，因为用户无法通过正常切换回到主界面。

需要权限：
```xml
<uses-permission android:name="android.permission.SYSTEM_ALERT_WINDOW" />
```

## 总结

| 问题 | 答案 |
|------|------|
| 为什么录音期间切不回主界面？ | 无障碍服务每次检测到窗口切换就启动OnePxActivity，然后强制模拟返回键 |
| 为什么只剩悬浮窗？ | 悬浮窗是系统级浮层，不触发窗口切换事件，不受返回键影响 |
| 为什么QQ在前台也能录到？ | VOICE_COMMUNICATION + 前台Activity(1像素) + 无障碍服务 + 前台通知，三重保活让系统允许访问音频流 |
| 为什么非华为设备不需要这些？ | 非华为系统对后台应用的VOICE_COMMUNICATION限制没那么严格 |

**核心结论**：通话录音Pro在荣耀设备上能成功，靠的不是单一技术，而是**无障碍服务 + 1像素Activity + performGlobalAction + 悬浮窗**形成的完整闭环系统。缺少任何一环都会失效。
