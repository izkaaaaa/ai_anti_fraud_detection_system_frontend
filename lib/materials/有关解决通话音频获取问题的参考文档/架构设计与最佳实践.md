## 核心模块设计

### 1. 无障碍服务模块（CallDetectionService）

**职责**：
- 监听系统事件
- 识别通话应用
- 触发录音流程

**关键方法**：
```
onAccessibilityEvent()
  ├─ 检测应用切换 (TYPE_WINDOW_STATE_CHANGED)
  ├─ 识别通话应用 (QQ/微信)
  ├─ 监听通话文本
  └─ 触发录音服务

onInterrupt()
  └─ 停止录音
```

**优点**：
- ✅ 系统级权限，可访问其他应用UI
- ✅ 后台持续运行
- ✅ 自动化触发，无需用户干预

**缺点**：
- ❌ 需要用户手动启用
- ❌ 某些设备可能限制
- ❌ 隐私敏感

---

### 2. 录音服务模块（RecordingService）

**职责**：
- 管理MediaRecorder生命周期
- 处理音频编码
- 保存录音文件

**关键流程**：
```
startRecording()
  ├─ 创建输出文件
  ├─ 初始化MediaRecorder
  ├─ 设置音频源 (MIC)
  ├─ 设置输出格式 (WAV/MP3)
  ├─ 设置编码器
  ├─ prepare()
  └─ start()

stopRecording()
  ├─ stop()
  ├─ release()
  └─ 保存元数据
```

**配置参数**：
```
AudioSource: MIC (麦克风)
OutputFormat: THREE_GPP / MPEG_4
AudioEncoder: AMR_NB / AAC
SamplingRate: 44100Hz / 48000Hz
Channels: 1 (单声道) / 2 (立体声)
BitRate: 128kbps - 256kbps
```

---

### 3. 保活服务模块（KeepAliveService）

**职责**：
- 创建前台通知
- 监听音频模式
- 保证后台运行

**保活策略**：
```
多层保活机制
├─ 前台服务 + 通知
│  └─ startForeground(id, notification)
├─ 1像素Activity (华为系统)
│  └─ 欺骗系统认为有前台Activity
├─ 广播接收器
│  └─ 监听系统广播，重启服务
└─ JobScheduler (可选)
   └─ 定时任务保活
```

**音频模式监听**：
```
AudioManager.MODE_IN_CALL
  └─ 通话中

AudioManager.MODE_IN_COMMUNICATION
  └─ 通信中

AudioManager.MODE_NORMAL
  └─ 正常模式
```

## 错误处理策略

### 异常场景处理

```
场景1: 麦克风被占用
├─ 检测: MediaRecorder.prepare() 失败
├─ 处理: 
│  ├─ 记录日志
│  ├─ 提示用户设置外放
│  └─ 重试机制 (延迟后重试)
└─ 降级: 使用AudioRecord替代

场景2: 存储空间不足
├─ 检测: IOException 或 ENOSPC
├─ 处理:
│  ├─ 停止录音
│  ├─ 清理旧文件
│  └─ 提示用户
└─ 降级: 删除最旧的录音

场景3: 无障碍服务被关闭
├─ 检测: onInterrupt() 被调用
├─ 处理:
│  ├─ 停止录音
│  ├─ 记录事件
│  └─ 提示用户重新启用
└─ 恢复: 用户手动重新启用

场景4: 应用被系统杀死
├─ 检测: onDestroy() 被调用
├─ 处理:
│  ├─ 保存当前状态
│  ├─ 释放资源
│  └─ 记录日志
└─ 恢复: 广播接收器重启服务
```

### 2. 数据加密

```kotlin
// 加密保存的录音文件
fun encryptAudioFile(inputFile: File, outputFile: File) {
    val cipher = Cipher.getInstance("AES/CBC/PKCS5Padding")
    val key = generateKey()
    cipher.init(Cipher.ENCRYPT_MODE, key)
    
    val inputStream = FileInputStream(inputFile)
    val outputStream = FileOutputStream(outputFile)
    
    val buffer = ByteArray(1024)
    var bytesRead: Int
    while (inputStream.read(buffer).also { bytesRead = it } != -1) {
        val encryptedData = cipher.update(buffer, 0, bytesRead)
        outputStream.write(encryptedData)
    }
    
    val finalData = cipher.doFinal()
    outputStream.write(finalData)
    
    inputStream.close()
    outputStream.close()
}
```

## 总结

### 关键设计原则

1. **模块化**：将功能分离为独立的Service
2. **可靠性**：多层保活机制确保后台运行
3. **隐私性**：加密存储，用户同意管理
4. **性能**：内存优化，电池优化
5. **可维护性**：清晰的代码结构，完善的日志
