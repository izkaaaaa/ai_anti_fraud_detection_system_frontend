# 通话录音Pro - 麦克风冲突解决方案技术分析

## 项目背景

本文档分析通话录音Pro应用如何在QQ/微信视频通话时，绕过系统麦克风占用限制，实现后台录音的技术方案。这对于AI诈骗识别应用具有重要参考价值。

---

## 核心问题分析

### 问题1：系统麦克风占用
- **现象**：QQ/微信视频通话时，这些应用作为高优先级隐私敏感源占用麦克风
- **原因**：Android系统对麦克风的独占性管理，同一时间只允许一个应用使用麦克风
- **传统方案失效**：直接调用 `AudioRecord` 获取麦克风数据会失败

### 问题2：隐私保护机制
- Android 6.0+ 对麦克风、摄像头等敏感资源进行严格管理
- 高优先级应用（如QQ、微信）获得麦克风独占权
- 其他应用无法同时访问麦克风

---

## 通话录音Pro的解决方案

### 方案核心思路：**曲线救国 - 音频重定向**

不直接获取系统麦克风，而是通过以下步骤：

```
QQ/微信通话 
  ↓
用户设置外放（扬声器播放对方声音）
  ↓
应用通过麦克风录制外放声音
  ↓
获得对方的音频数据
```

**关键点**：当对方声音通过扬声器外放时，应用可以通过麦克风"听到"这个声音，从而绕过系统的麦克风占用限制。

### 1. 无障碍服务（AccessibilityService）- 核心触发机制

**文件**：`RecordHelperService.java`

#### 作用：
- 监听系统事件和应用切换
- 检测通话状态

#### 为什么需要无障碍服务？
1. **系统事件监听**：无障碍服务可以监听系统级别的事件，包括应用切换、文本变化等
2. **权限突破**：无障碍服务拥有特殊权限，可以访问其他应用的UI信息
3. **后台运行**：即使应用在后台，无障碍服务仍可继续运行
4. **自动化触发**：无需用户手动操作，自动检测通话并触发录音

### 2. 华为/荣耀设备特殊处理 - 1像素Activity

**文件**：`RecordHelperService.java` 和 `OnePXActivity.java`

#### 问题背景：
华为/荣耀系统对后台应用的限制更严格，直接启动服务可能被系统杀死。

#### 解决方案：

```java
String str = Build.BRAND;
if (TextUtils.equals(str.toLowerCase(), "huawei") || 
    TextUtils.equals(str.toLowerCase(), "honor")) {
    
    // 启动1像素透明Activity
    Intent intent = new Intent(this, OnePXActivity.class);
    if (last != null) {
        last.startActivity(intent);
    } else {
        intent.setFlags(268435456);  // FLAG_NEW_TASK
        startActivity(intent);
    }
}
```

#### 1像素Activity的作用：
1. **欺骗系统**：系统认为应用有前台Activity在运行
2. **保活机制**：Activity存在时，应用进程优先级提高，不易被杀死
3. **隐蔽性**：1像素大小，用户几乎看不到
4. **后台录音**：Activity存在期间，后台服务可以继续运行

#### 配置示例：

### 3. 前台服务保活 - KeepAliveService

**文件**：`KeepAliveService.java`

#### 作用：
- 创建前台通知，提高应用进程优先级
- 监听音频模式变化，检测通话状态
- 确保应用在后台持续运行

#### 关键实现：

```java
@Override
public int onStartCommand(Intent intent, int i, int i2) {
    // 1. 创建前台通知
    NotificationChannel notificationChannel = 
        new NotificationChannel("KeepAlive", "通话录音Pro保活服务", 4);
    notificationChannel.setSound(null, null);
    
    // 2. 构建通知
    Notification.Builder builder = new Notification.Builder(this, "KeepAlive");
    builder.setContentTitle("通话录音Pro")
           .setContentText("正在监听通话...")
           .setSmallIcon(R.drawable.icon_notifation)
           .setOngoing(true);  // 用户无法滑动关闭
    
    // 3. 启动前台服务
    startForeground(20240501, builder.build());
    
    // 4. 监听音频模式变化
    if (Build.VERSION.SDK_INT >= 31) {
        AudioManager audioManager = 
            (AudioManager) getSystemService("audio");
        audioManager.addOnModeChangedListener(
            getMainExecutor(), 
            new OooO0OO(this)  // 监听器
        );
    } else {
        // 低版本使用轮询
        Executors.newSingleThreadExecutor().submit(
            new OooO0o(this, audioManager)
        );
    }
    
    return 2;  // START_STICKY - 服务被杀死后自动重启
}
```

#### 音频模式监听的意义：
- `AudioManager.MODE_IN_CALL`：通话中
- `AudioManager.MODE_IN_COMMUNICATION`：通信中
- 当检测到这些模式变化时，触发录音逻辑

## 完整工作流程

```
应用启动
  ↓
SplashActivity 检查设备品牌
  ├─ 非华为/荣耀 → 启动 KeepAliveService
  └─ 华为/荣耀 → 延迟启动（在首次通话时启动）
  ↓
KeepAliveService 启动
  ├─ 创建前台通知（保活）
  ├─ 监听音频模式变化
  └─ 启动 RecordHelperService（无障碍服务）
  ↓
RecordHelperService 运行
  ├─ 监听应用切换事件
  ├─ 识别 QQ/微信 视频通话
  └─ 监听通话邀请文本
  ↓
检测到通话
  ├─ 提取通话对方信息
  ├─ 华为/荣耀设备 → 启动 OnePXActivity（1像素）
  └─ 触发 RecordService 开始录音
  ↓
用户设置外放
  ├─ 对方声音通过扬声器播放
  └─ 应用麦克风录制外放声音
  ↓
录音完成
  ├─ 保存 RecordEntity（元数据）
  ├─ 保存音频文件（WAV/PCM/MP3）
  └─ 用户可在 MainActivity 查看和播放
```

## 关键技术点总结

| 技术点 | 作用 | 原理 |
|------|------|------|
| **无障碍服务** | 监听系统事件，自动触发录音 | 系统级权限，可访问其他应用UI |
| **前台服务** | 保活应用进程 | 创建通知，提高进程优先级 |
| **1像素Activity** | 华为系统保活 | 欺骗系统认为有前台Activity |
| **音频模式监听** | 检测通话状态 | AudioManager 监听模式变化 |
| **麦克风录制** | 获取外放声音 | 通过麦克风录制环境中的声音 |
| **广播接收器** | 系统事件触发 | 监听系统广播，启动服务 |

## 潜在问题和解决方案

### 问题1：麦克风录制的音质问题

**现象**：通过麦克风录制的外放声音质量较低

**解决方案**：
- 使用高采样率（48000Hz）
- 使用立体声录制
- 后处理：降噪、增强

```java
// 配置高质量录音
mediaRecorder.setAudioSamplingRate(48000);
mediaRecorder.setAudioChannels(2);  // 立体声
mediaRecorder.setAudioEncodingBitRate(256000);
```

### 问题2：系统更新导致兼容性问题

**解决方案**：
- 针对不同Android版本采用不同方案
- 定期测试新系统版本
- 提供降级方案

```java
if (Build.VERSION.SDK_INT >= 31) {
    // Android 12+ 方案
    useAudioManagerListener();
} else if (Build.VERSION.SDK_INT >= 29) {
    // Android 10-11 方案
    usePollingMethod();
} else {
    // Android 9 及以下方案
    useLegacyMethod();
}
```

### 问题3：用户隐私和权限

**解决方案**：
- 清晰的权限说明
- 用户可随时关闭
- 数据本地存储，不上传

```java
// 权限请求
if (Build.VERSION.SDK_INT >= 23) {
```

## 总结

通话录音Pro通过以下创新方案解决了麦克风冲突问题：

1. **无障碍服务**：自动检测通话，无需用户手动操作
2. **前台服务**：保证后台持续运行
3. **1像素Activity**：针对华为系统的特殊处理
4. **音频重定向**：通过外放 + 麦克风录制获得对方声音

这套方案对于AI诈骗识别应用具有重要参考价值，但需要注意：
- ⚠️ 用户隐私保护
- ⚠️ 法律合规性
- ⚠️ 系统兼容性
- ⚠️ 用户体验

建议在实现时采用**分层方案**：
- 优先使用系统API（如果可用）
- 降级到无障碍服务方案
- 最后降级到麦克风录制方案

这样可以在保证功能的同时，最大化用户体验和系统兼容性。
