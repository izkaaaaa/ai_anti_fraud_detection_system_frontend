# 通话录音问题修复建议

> 基于对通话录音Pro源码的逆向分析，找到了我们Flutter项目录不到通话声音的根本原因。

---

## 结论先行

**我们之前用错了声音源。**

通话录音Pro在通话录音时用的是 `AudioSource.VOICE_COMMUNICATION`（值为6），而不是 `AudioSource.MIC`（值为1）。

---

## 源码证据

在通话录音Pro的录音器初始化代码中（`OooOO0O.java`）：

```java
// 录音配置初始化，第一个参数 6 = VOICE_COMMUNICATION
public OooOOO0 f7644OooOOOO = new OooOOO0(6, 16000, 16, 2);

// 最终传给 AudioRecord.Builder
builder.setAudioSource(this.f7518OooO00o.f7526OooO00o);  // = 6
```

Android官方 `AudioSource` 常量对照：

| 值 | 常量名 | 说明 |
|----|--------|------|
| 1 | `MIC` | 普通麦克风 |
| 6 | `VOICE_COMMUNICATION` | 专为VoIP通话设计，针对回声消除做了优化 |
| 7 | `VOICE_RECOGNITION` | 语音识别 |

---

## 为什么这很关键

QQ/微信视频通话时，系统的音频路由是这样的：

```
对方声音 → 系统通话音频流（VOICE_COMMUNICATION 通道）→ 扬声器播放
```

- `VOICE_COMMUNICATION` 是系统通话的专用通道，可以直接读取到通话中双方的声音
- `MIC` 只能录制物理麦克风采集到的环境声音（外放时才能听到对方）
- 荣耀/华为系统在通话时会对后台的 `MIC` 请求做额外封锁，但 `VOICE_COMMUNICATION` 有不同的处理策略

---

## 需要修改的地方

找到我们项目原生 Android 端的录音初始化代码，修改 `AudioSource`：

### 修改前（错误）

```kotlin
// Kotlin
val audioRecord = AudioRecord(
    MediaRecorder.AudioSource.MIC,  // ← 错误，改这里
    16000,
    AudioFormat.CHANNEL_IN_MONO,
    AudioFormat.ENCODING_PCM_16BIT,
    bufferSize
)
```

或者用 Builder 方式：

```kotlin
val audioRecord = AudioRecord.Builder()
    .setAudioSource(MediaRecorder.AudioSource.MIC)  // ← 错误，改这里
    .setAudioFormat(...)
    .setBufferSizeInBytes(bufferSize)
    .build()
```

### 修改后（正确）

```kotlin
// Kotlin
val audioRecord = AudioRecord(
    MediaRecorder.AudioSource.VOICE_COMMUNICATION,  // ← 改成这个
    16000,
    AudioFormat.CHANNEL_IN_MONO,
    AudioFormat.ENCODING_PCM_16BIT,
    bufferSize
)
```

或者用 Builder 方式：

```kotlin
val audioRecord = AudioRecord.Builder()
    .setAudioSource(MediaRecorder.AudioSource.VOICE_COMMUNICATION)  // ← 改成这个
    .setAudioFormat(
        AudioFormat.Builder()
            .setSampleRate(16000)
            .setChannelMask(AudioFormat.CHANNEL_IN_MONO)
            .setEncoding(AudioFormat.ENCODING_PCM_16BIT)
            .build()
    )
    .setBufferSizeInBytes(bufferSize)
    .build()
```

如果用的是 Java：

```java
AudioRecord audioRecord = new AudioRecord(
    MediaRecorder.AudioSource.VOICE_COMMUNICATION,  // ← 改成这个，值为 6
    16000,
    AudioFormat.CHANNEL_IN_MONO,
    AudioFormat.ENCODING_PCM_16BIT,
    bufferSize
);
```

## 录音参数参考

通话录音Pro使用的参数如下，建议我们保持一致：

| 参数 | 值 | 说明 |
|------|-----|------|
| AudioSource | 6 (VOICE_COMMUNICATION) | 通话专用声音源 |
| 采样率 | 16000 Hz | 16kHz，语音识别常用 |
| 位深 | 16 bit | PCM_16BIT |
| 声道 | 单声道 | CHANNEL_IN_MONO |

## 如果改了还是不行

`VOICE_COMMUNICATION` 本身也可能被QQ/微信占用（和MIC类似），这时需要同时满足以下条件：

### 条件1：无障碍服务必须开启 ✅（你们已经做了）

无障碍服务赋予了应用系统级别的权限，这是绕过限制的关键前提。

### 条件2：前台服务通知必须可见

确认通知栏有我们的应用通知，说明前台服务在运行：

```kotlin
startForeground(NOTIFICATION_ID, buildNotification())
```

### 条件3：华为/荣耀设备需要启动1像素Activity

这是通话录音Pro针对华为系统的专属处理，代码逻辑是：

```kotlin
val brand = Build.BRAND.lowercase()
if (brand == "huawei" || brand == "honor") {
    // 通话开始时，启动1像素透明Activity
    val intent = Intent(this, OnePxActivity::class.java)
    intent.flags = Intent.FLAG_ACTIVITY_NEW_TASK
    startActivity(intent)
}
```

`OnePxActivity` 的主题配置（`AndroidManifest.xml`）：

```xml
<activity
    android:name=".OnePxActivity"
    android:theme="@style/OnePxActivityStyle"
    android:launchMode="singleInstance"
    android:exported="false" />
```

`res/values/styles.xml` 中添加：

```xml
<style name="OnePxActivityStyle" parent="AppTheme">
    <item name="android:windowIsTranslucent">true</item>
    <item name="android:windowBackground">@android:color/transparent</item>
    <item name="android:windowContentOverlay">@null</item>
    <item name="android:windowNoTitle">true</item>
    <item name="android:windowIsFloating">true</item>
    <item name="android:backgroundDimEnabled">false</item>
</style>
```

`OnePxActivity.kt`：

```kotlin
class OnePxActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        // 设置窗口为1像素
        window.setLayout(1, 1)
        window.setGravity(Gravity.LEFT or Gravity.TOP)
        val params = window.attributes
        params.x = 0
        params.y = 0
        window.attributes = params
    }
}
```

## 建议的测试步骤

1. **先只改声音源**，把 `MIC` 改成 `VOICE_COMMUNICATION`，在荣耀手机上测试QQ通话
2. 如果还是不行，再**加上1像素Activity**，在通话被检测到时启动它
3. 确认前台服务通知是否正常显示
4. 对比通话录音Pro和我们app的日志，看看哪一步失败了

---

## 补充说明

通话录音Pro能成功的核心组合是：

```
VOICE_COMMUNICATION 声音源
    + 无障碍服务（监听通话开始）
    + 前台服务（保活）
    + 1像素Activity（华为系统保活）
```

缺少任何一个环节都可能导致在荣耀手机上录不到声音。
