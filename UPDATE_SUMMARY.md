# 前端更新总结 - 用户画像功能

## 📋 更新概述

根据后端新版本 API，前端已完成用户画像字段的全面支持，包括注册、显示和更新功能。

---

## ✅ 已完成的更新

### 1. 数据模型更新

**文件**: `lib/viewmodels/login_models.dart`

- ✅ `User` 模型新增字段：
  - `roleType` - 角色类型（青壮年/老人/学生/其他）
  - `gender` - 性别（男/女/未知）
  - `profession` - 职业
  - `maritalStatus` - 婚姻状况（单身/已婚/离异）

- ✅ `RegisterRequest` 模型支持新字段（可选）

### 2. API 接口更新

**文件**: `lib/api/auth_api.dart`

- ✅ `registerAPI` - 支持传递用户画像字段
- ✅ `updateUserProfileAPI` - 新增用户画像更新接口

### 3. 注册页面更新

**文件**: `lib/pages/Register/index.dart`

- ✅ 新增"完善资料"折叠区域
- ✅ 添加表单字段：
  - 角色类型（下拉选择）
  - 性别（下拉选择）
  - 职业（文本输入）
  - 婚姻状况（下拉选择）
- ✅ 所有新字段均为可选，不影响基本注册流程
- ✅ 添加"推荐"标签提示用户完善资料

### 4. 个人中心页面更新

**文件**: `lib/pages/Profile/index.dart`

- ✅ 用户信息卡片显示新字段（如果已填写）
- ✅ 新增"完善资料"菜单项（高亮显示）
- ✅ 新增 `EditProfilePage` 完善资料页面
  - 支持更新所有用户画像字段
  - 实时保存到后端
  - 自动刷新用户信息

### 5. 配置更新

**文件**: `lib/contants/index.dart`

- ✅ 超时时间从 10 秒增加到 60 秒
- ✅ 支持 LLM 生成安全报告的长时间请求

## 🔧 技术实现

### 字段映射

| 前端字段 | 后端字段 | 类型 | 可选 |
|---------|---------|------|------|
| `roleType` | `role_type` | String | ✅ |
| `gender` | `gender` | String | ✅ |
| `profession` | `profession` | String | ✅ |
| `maritalStatus` | `marital_status` | String | ✅ |

### API 调用

```dart
// 注册时传递画像字段
await registerAPI(
  phone: phone,
  username: username,
  name: name,
  password: password,
  smsCode: smsCode,
  roleType: '青壮年',
  gender: '男',
  profession: '工程师',
  maritalStatus: '已婚',
);

// 更新用户画像
await updateUserProfileAPI(
  roleType: '老人',
  gender: '女',
  profession: '退休教师',
  maritalStatus: '已婚',
);
```

## 📝 注意事项

1. **字段值规范**
   - 性别：只能是"男"、"女"、"未知"
   - 婚姻状况：只能是"单身"、"已婚"、"离异"
   - 角色类型：只能是"青壮年"、"老人"、"学生"、"其他"
   - 职业：自由输入，建议提供常见职业列表

2. **数据验证**
   - 前端使用下拉框限制选项
   - 后端应该也做相应验证

3. **隐私保护**
   - 这些字段都是敏感信息
   - 建议在隐私政策中说明用途
