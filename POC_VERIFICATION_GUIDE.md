# 音频录制 POC 验证指南

## 概述

本指南说明如何验证 VOICE_COMMUNICATION 音频录制方案是否可行。

## 验证步骤

### 第一步：打开测试页面

在你的 Flutter 应用中添加路由，打开测试页面：

```dart
// 在你的路由配置中添加
import 'package:ai_anti_fraud_detection_system_frontend/pages/AudioRecordingTestPage.dart';

// 在路由表中添加
'/audio-test': (context) => AudioRecordingTestPage(),
```

或者直接在 main.dart 中导航：

```dart
Navigator.pushNamed(context, '/audio-test');
```

### 第二步：启动应用

```bash
flutter run
```

### 第三步：进行测试

#### 测试场景 1：微信通话

1. 打开测试页面
2. 点击"启动录制"
3. 打开微信，进行视频通话
4. 观察日志输出

**预期结果：**
```
✅ 音频录制已启动
🎤 使用音频源: VOICE_COMMUNICATION
✅ 使用 VOICE_COMMUNICATION - 可以与微信/QQ 共享麦克风
🎤 收到音频数据: 1024 bytes
🎤 收到音频数据: 1024 bytes
...
```

#### 测试场景 2：QQ 通话

1. 点击"停止录制"
2. 打开 QQ，进行视频通话
3. 点击"启动录制"
4. 观察日志输出

**预期结果：**
```
✅ 音频录制已启动
🎤 使用音频源: VOICE_COMMUNICATION 或 MIC
🎤 收到音频数据: 1024 bytes
...
```

#### 测试场景 3：打开免提

1. 在微信/QQ 通话中打开免提
2. 观察是否能获取到对方的声音

**预期结果：**
```
🎤 收到音频数据: 1024 bytes (应该包含对方的声音)
```

### 第四步：查看详细日志

打开 Android Studio 的 Logcat，过滤 "AudioRecording"：

```bash
adb logcat | grep "AudioRecording"
```

**关键日志：**

```
✅ Recording started with source: VOICE_COMMUNICATION
🎤 Read audio data: 1024 bytes
```

## 验证清单

### ✅ 成功标志

- [ ] 应用启动无崩溃
- [ ] 点击"启动录制"后，日志显示 "✅ 音频录制已启动"
- [ ] 日志显示音频源为 "VOICE_COMMUNICATION"

- [ ] 音频帧数和总字节数不断增加

### ⚠️ 降级标志

- [ ] 日志显示音频源为 "MIC"（说明 VOICE_COMMUNICATION 不可用）
- [ ] 需要打开免提才能获取对方声音
- [ ] 这是正常的降级行为

### ❌ 失败标志

- [ ] 应用崩溃
- [ ] 日志显示错误信息
- [ ] 无法启动录制
- [ ] 无法获取任何音频数据

## 关键指标

### 1. 音频源类型

```
VOICE_COMMUNICATION ✅ 最好
  ↓
MIC ⚠️ 需要打开免提
  ↓
VOICE_RECOGNITION ⚠️ 备选方案
  ↓
UNKNOWN ❌ 失败
```

### 2. 音频数据量

**正常情况：**
- 每秒应该收到约 15-20 个音频帧
- 每帧约 1024-2048 bytes
- 每秒约 15-40 KB

**异常情况：**
- 没有收到任何数据（分贝为 0）
- 数据量很小（< 1 KB/s）

### 3. 音频源变化

**微信通话：**
```
VOICE_COMMUNICATION → 正常
MIC → 需要打开免提
```

**QQ 通话：**
```
VOICE_COMMUNICATION → 正常
MIC → 需要打开免提
```

## 故障排查

### 问题 1：无法启动录制

**症状：**
```
❌ 启动失败
```

**可能原因：**
1. 没有授予 RECORD_AUDIO 权限
2. 麦克风被其他应用独占
3. 设备不支持

**解决方案：**
1. 检查权限：设置 → 应用 → 权限 → 麦克风
2. 关闭其他使用麦克风的应用
3. 重启设备

### 问题 2：收不到音频数据

**症状：**
```
✅ 音频录制已启动
🎤 使用音频源: VOICE_COMMUNICATION
（但没有 "收到音频数据" 的日志）
```

**可能原因：**
1. 麦克风没有声音
2. 音频源不支持
3. 系统限制

**解决方案：**
1. 确保有声音输入（说话或播放音乐）
2. 检查音频源是否为 MIC（需要打开免提）
3. 尝试其他应用（微信、QQ 等）

### 问题 3：音频源为 MIC

**症状：**
```
🎤 使用音频源: MIC
⚠️ 已降级到 MIC - 只能录本地声音，需要打开免提
```

**原因：**
- VOICE_COMMUNICATION 在该设备/应用上不可用
- 这是正常的降级行为

**解决方案：**
1. 打开免提，让对方声音通过扬声器播放
2. 或者改变产品设计

### 问题 4：应用崩溃

**症状：**
```
应用闪退或无响应
```

**可能原因：**
1. 权限问题
2. 内存不足
3. 代码异常

**解决方案：**
1. 查看 logcat 错误日志
2. 检查 AndroidManifest.xml 权限声明
3. 重新编译项目

## 常见问题

**Q: 为什么在 QQ 通话时收不到音频？**

A: 可能是以下原因：
1. QQ 比微信更严格地独占麦克风
2. 已自动降级到 MIC，需要打开免提
3. 设备不支持 VOICE_COMMUNICATION

**Q: 如何确认是否真的在使用 VOICE_COMMUNICATION？**

A: 查看日志中的 "使用音频源" 字段。

**Q: 为什么音频数据为 0？**

A: 可能是：
1. 没有声音输入
2. 麦克风被占用
3. 系统限制

**Q: 如何提高音频质量？**

A: 可以调整采样率、缓冲大小等参数（见 AudioRecordingService.kt）。
