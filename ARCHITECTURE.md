## 🔄 数据流

### 1. 音频流

```
QQ/微信 通话
    ↓
VOICE_COMMUNICATION 音频源
    ↓
AudioRecordingService (Kotlin)
    ├─ 采样率：16000 Hz
    ├─ 位深：16 bit
    ├─ 声道：单声道
    └─ 每帧：1280 bytes
    ↓
Platform Channel
    ↓
AudioRecordingServiceDart (Dart)
    ↓
_audioBuffer (缓冲区)
    ├─ 缓冲大小：16000 bytes (1秒)
    └─ 发送间隔：1秒
    ↓
WebSocket
    ├─ 消息类型：audio
    ├─ 编码：Base64
    └─ 大小：~16 KB/s
    ↓
后端音频检测
    ├─ 语音特征提取
    ├─ 诈骗语音识别
    └─ 情感分析
    ↓
检测结果
    ├─ 风险等级
    ├─ 置信度
    └─ 建议措施
```

### 2. 视频流

```
屏幕内容（对方人像）
    ↓
MediaProjection (Kotlin)
    ├─ 分辨率：设备分辨率
    ├─ 格式：JPEG
    └─ 帧率：1 fps (可动态调整)
    ↓
ScreenCaptureService (Kotlin)
    ↓
Platform Channel
    ↓
RealTimeDetectionService (Dart)
    ↓
WebSocket
    ├─ 消息类型：video
    ├─ 编码：Base64
    └─ 大小：~40 KB/frame
    ↓
后端视频检测
    ├─ 人脸识别
    ├─ 异常行为检测
    └─ 深度伪造检测
    ↓
检测结果
    ├─ 风险等级
    ├─ 置信度
    └─ 建议措施
```

### 3. 文字流

```
对方声音
    ↓
百度语音识别 (WebSocket)
    ├─ 采样率：16000 Hz
    ├─ 格式：PCM 16bit
    └─ 实时识别
    ↓
识别结果
    ├─ 临时结果 (MID_TEXT)
    └─ 最终结果 (FIN_TEXT)
    ↓
RealTimeDetectionService (Dart)
    ↓
WebSocket
    ├─ 消息类型：text
    ├─ 内容：识别的文本
    └─ 大小：~1 KB/s
    ↓
后端文本检测
    ├─ 关键词匹配
    ├─ NLP 分析
    └─ 诈骗文本识别
    ↓
检测结果
    ├─ 风险等级
    ├─ 置信度
    └─ 建议措施
```

## 🛡️ 三级防御机制

### Level 1: 正常模式（绿色）

```
状态：正常监测中
特征：
  - 视频帧率：1 fps
  - 检测频率：正常
  - 通知方式：定时通知（每10秒）
  - 用户感知：低

操作：
  - 继续监测
  - 定期提醒用户
  - 收集数据用于学习
```

### Level 2: 警惕模式（黄色）

```
状态：检测到可疑行为
特征：
  - 视频帧率：2-3 fps
  - 检测频率：提高
  - 通知方式：中风险警告
  - 用户感知：中等

操作：
  - 提高检测频率
  - 发送中风险警告
  - 开启通话录音
  - 建议用户提高警惕
```

### Level 3: 危险模式（红色）

```
状态：检测到高风险诈骗
特征：
  - 视频帧率：5+ fps
  - 检测频率：最高
  - 通知方式：高风险警告（全屏）
  - 用户感知：高

操作：
  - 最高检测频率
  - 发送高风险警告
  - 强制开启通话录音
  - 建议用户立即挂断
```
